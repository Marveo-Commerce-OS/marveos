# Known Limitations

## Current MVP Limits

- No full sync engine yet.
- No Vercel/GitHub deployment automation yet.
- No real payment integration yet.
- Support officer matching is still placeholder queue routing.
- Client-only auth test identity is still needed for complete portal-separation proof.
- Template catalog is still placeholder-level, not a finalized commercial catalog.
- Shopify, Laravel, and generic API connectors are upcoming, not current MVP-ready.

## Demo Guidance

- Present this MVP as guided onboarding and operational visibility, not full production automation.
- Do not promise automatic deployment pipelines.
- Do not promise full commerce-platform parity beyond the current WordPress connector path.
- Do not present billing summary surfaces as live payment operations.

## Safe Positioning

What is ready to show:
- internal master access
- guided setup MVP
- existing website verification flow
- connector status persistence
- launch checklist and readiness views

What is not ready to position as complete:
- production sync/import workflows
- automated infra deployment
- live billing operations
- finished client-role demo identity coverage
