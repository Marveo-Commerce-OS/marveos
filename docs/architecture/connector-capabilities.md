# Connector Capabilities (Phase 2F Audit)

This document captures the connector architecture audit performed during Phase 2F, mapping what the Marvéo Connector plugin exposes and what MarvéoOS can safely use.

---

## Connector Plugin Overview

**Plugin:** `marveo-connector` v1.0.16  
**Namespace:** `marveo/v1` (all endpoints under `/wp-json/marveo/v1/`)  
**Location:** `marveo-connector/` repository

---

## Endpoint Map and Capability Status

### Public Endpoints (No Authentication Required)

| Endpoint | Method | Auth | Status | Used by MarvéoOS |
|---|---|---|---|---|
| `/marveo/v1/status` | GET | None | **Working** | Yes — connector check + verify |

**`/status` returns:**
- `status` — connector plugin status (`pending_setup`, `active`, `disconnected`)
- `connector_version` — plugin version string
- `site_id` — unique site identifier generated at activation
- `wordpress_version` — WP version
- `woocommerce_version` — WC version or null if not installed
- `jwt_enabled` — whether JWT Auth plugin is active
- `first_admin_created` — whether onboarding admin was created
- `deployment_status` — current deployment readiness profile
- `deployment_profile` — full deployment profile object
- `setup_completed` — boolean
- `validation_passed` — boolean

## Token Architecture

| Token | Purpose | Generated | Lifespan |
|---|---|---|---|
| `marveo_activation_token` | Creates first WP admin (`/init-admin`) | On plugin activation | 24 hours |
| `marveo_command_token` | Authorizes command endpoints | Stored WP option or env var | Persistent |
| Workspace connector token (MarvéoOS-side) | Identifies workspace in connection flow | Generated by MarvéoOS | Persistent until reset |

## Connection Verification Flow (Phase 2F)

The MVP existing website connection uses the following server-side proxy sequence:

1. **MarvéoOS generates workspace connector token** (`POST /api/cloud/workspaces/:id/connector`)
   - Workspace state: `TOKEN_GENERATED`

2. **Client installs and activates the plugin** on their WordPress site
   - Plugin generates `marveo_activation_token` and sets status to `pending_setup`

3. **Client pastes workspace connector token + domain in onboarding wizard**
   - Stored in `collectedBusinessData.connectorToken`

4. **"Verify connection" button** calls `POST /api/cloud/connector/preflight` (server-side proxy)
   - Calls `GET {domain}/wp-json/marveo/v1/status`
   - Calls runtime-safe endpoints to derive capabilities and counts
   - Returns plugin presence, WP/WC versions, connector status, and metadata
   - No workspace required — used for real-time onboarding feedback

5. **On deployment**, `POST /api/cloud/workspaces/:id/connector/verify` is called (non-blocking)
   - Calls `GET {domain}/wp-json/marveo/v1/status` (public)
   - Calls `GET {domain}/wp-json/marveo/v1/site-profile` (runtime permission)
   - Optionally validates connector token match
   - Stores full `connectorSiteMetadata` in workspace
   - Sets `connectorStatus` to `CONNECTED` or `FAILED`

---

## Workspace Connector State

| State | Meaning |
|---|---|
| `NOT_CONNECTED` | Default. No connection attempt made. |
| `TOKEN_GENERATED` | MarvéoOS has generated a workspace invite token. |
| `PENDING_VERIFICATION` | Verify endpoint was called, awaiting result. |
| `CONNECTED` | Plugin detected and site metadata stored. |
| `FAILED` | Verification attempt failed. Error stored. |
| `SUPPORT_REQUIRED` | Manual setup route selected or connection could not be established. |

---

## Stored Site Metadata (on CONNECTED)

```typescript
{
  siteUrl: string;
  siteName?: string;
  platform: string;            // "WordPress/WooCommerce" or "WordPress (Headless)"
  wordpressVersion?: string;
  woocommerceEnabled?: boolean;
  connectorVersion?: string;
  connectorPluginStatus?: string; // "pending_setup" | "active"
  siteId?: string;
  jwtEnabled?: boolean;
  discoveredAt: string;         // ISO timestamp
}
```

## What Is Working (Phase 2F)

- Public status endpoint reachable via server-side proxy
- Site profile fetch (no user auth needed, only connector status check)
- Plugin presence detection and WP/WC version discovery
- Connector token generation per workspace
- Connection state transitions with audit logging
- Onboarding wizard "Check plugin" real-time feedback
- Deployment-phase connector verify (non-blocking)
- Launch checklist connector_verified item for EXISTING_WEBSITE workspaces
- Internal dashboard shows connectorStatus and connectorSiteMetadata

---

## What Is Partial (Phase 2F)

- `/marveo/v1/site-info` — requires JWT auth; JWT plugin not always installed → not wired
- `/marveo/v1/content-inventory` — requires admin session → available via command flow only
- `/marveo/v1/content-scan` — same as above
- `/marveo/v1/settings`, `/marveo/v1/brand`, etc. — available at runtime permission level; not yet fetched in connection flow

---

## What Is Placeholder (Phase 2F)

- No real activation token exchange between WP plugin and MarvéoOS
- No handshake or mutual verification with activation token (token is stored client-side only)
- No content pull, sync engine, or media import
- No WooCommerce product count or content sampling from connector

---

## Recommended Phase 3

1. **Activation token handshake:** Exchange the WP `marveo_activation_token` with MarvéoOS for secure mutual verification.
2. **Lightweight content discovery:** After connection, call `/marveo/v1/content-inventory` (via admin token flow) to display pages/products counts in workspace detail.
3. **Content sample pull:** Fetch first page of pages and products for workspace preview.
4. **JWT integration:** Guide admin to install JWT Auth plugin for full site-info access.
5. **Connection health monitoring:** Periodic re-verification of `connectorStatus` for active workspaces.
6. **WooCommerce-specific onboarding path:** Separate branch for WC-enabled sites with product catalog linking.
